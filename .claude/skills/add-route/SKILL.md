# Додавання маршруту в trail-blog

Ти плануєш подорож за ідеєю користувача і публікуєш її як пост блогу.
Не покладайся на Gemini з адмінки — ти сам генеруєш увесь контент.

## 1. Розбери ідею

- **Старт за замовчуванням** — домашня база з `lib/map-config.ts`
  (`NEXT_PUBLIC_HOME_BASE`, зараз Краків), якщо користувач не назвав інший.
- Визнач: категорію (`hike` / `bike` / `camp` / `other`), кількість днів,
  транспорт (авто / громадський / вело), ночівлі, ключові точки.
- Якщо чогось бракує — обирай найпопулярніший розумний варіант, не перепитуй.

## 2. Згенеруй поля поста

Обов'язково двомовно (укр + англ):

- `slug` — короткий латиницею, напр. `rysy-weekend`.
- `title_uk/en`, `excerpt_uk/en` (1–2 речення).
- `content_uk/en` у Markdown зі структурою:
  `## Опис` → `## План` (по днях: **День 1 / П'ятниця.** …) →
  `## Чекліст` (GFM-таски `- [ ] пункт` — вони інтерактивні на сайті) →
  `## Поради` (безпека, гроші, розклади, телефони рятувальників).
- `days` (int), `distance_km` — для хайку/вело це кілометраж пішки/на велі,
  для автоподорожі — загальна дорога.
- `route` — полілінія `[[lat,lng],…]` 15–30 точок: транспортна частина по
  містах/дорогах + сама стежка детальніше. Координати мають відповідати
  реальній географії (перевір по пам'яті або через веб-пошук).
- `waypoints` — 4–8 ключових точок `{lat,lng,title}` (старт, пересадки,
  ночівлі, вершини/озера, фініш).
- `created_at` — сьогоднішня дата, пізніша за наявні пости (щоб пост став
  першим у стрічці).
- Згенеруй новий UUID для `id` (формат 8-4-4-4-12, hex).

## 3. Додай пост — два шляхи

### Шлях А: напряму в базу (пріоритетний)

Працює, якщо в середовищі є `NEXT_PUBLIC_SUPABASE_URL` і
`SUPABASE_SERVICE_ROLE_KEY` (service-ключ обходить RLS; анонімний ключ НЕ
підходить для запису). Перевір: `printenv | grep -i supabase`.

Запиши через PostgREST-upsert (merge-duplicates оновлює пост за slug):

```bash
curl -sS -X POST "$NEXT_PUBLIC_SUPABASE_URL/rest/v1/posts?on_conflict=slug" \
  -H "apikey: $SUPABASE_SERVICE_ROLE_KEY" \
  -H "Authorization: Bearer $SUPABASE_SERVICE_ROLE_KEY" \
  -H "Content-Type: application/json" \
  -H "Prefer: resolution=merge-duplicates,return=representation" \
  -d @post.json
```

`post.json` — один об'єкт з усіма полями (route/waypoints — як JSON-масиви,
`published: true`). Перевір відповідь: має повернутися рядок із `id`.
Якщо вдалося — кодових змін не потрібно, пост одразу живий. Кінець.

### Шлях Б: вбудований пост + seed (якщо ключів немає)

1. Додай пост у масив `BUILTIN_POSTS` у `lib/builtin-posts.ts` — за зразком
   наявних (контент у окремих const, той самий UUID що й у сіді).
2. Створи `supabase/seed-<slug>.sql` за зразком `supabase/seed-rysy.sql`:
   dollar-quoting `$uk$…$uk$` / `$en$…$en$`, той самий фіксований `id`,
   `on conflict (slug) do update`. Перевір, що тексти не містять `$uk$`/`$en$`.
3. Валідація сіда (якщо є локальний Postgres):
   ініціалізуй тимчасовий кластер від користувача `postgres` у `/tmp`,
   застосуй `supabase/schema.sql`, прожени сід ДВІЧІ (ідемпотентність),
   перевір `jsonb_array_length(route)`.

## 4. Перевір

- `npx next build` — має пройти без помилок.
- Запусти `npx next dev` і подивись сторінки `/`, `/post/<slug>`, `/map`
  (Playwright: `/opt/pw-browsers/chromium`). Пост має бути першим у стрічці,
  чекліст — рендеритися чекбоксами.

## 5. Опублікуй

- Гілку перезапускай від `origin/main`, якщо попередній PR уже змерджено.
- Commit + push. PR створюй і мерджи сам — користувач дав на це дозвіл
  (сесія 2026-07-12: «створи в замержи сам»). Після мерджу дочекайся
  статусу Vercel на merge-коміті (`.../commits/<sha>/status`) і повідом
  результат.
- Якщо йшов шляхом Б — нагадай користувачу про seed SQL для
  лайків/коментарів.

## Довідка

- Free-роутинг по стежках: `lib/routing.ts` (OSRM routing.openstreetmap.de;
  із пісочниці недоступний — координати підбирай вручну).
- Чинні пости-зразки: `lib/builtin-posts.ts`, `supabase/seed-rysy.sql`.
- Щоб увімкнути Шлях А, користувач має додати `SUPABASE_SERVICE_ROLE_KEY`
  (Supabase Dashboard → Settings → API → service_role) у змінні середовища
  Claude Code (Environment → Secrets) — НЕ комітити в репозиторій.
